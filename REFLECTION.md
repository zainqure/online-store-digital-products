# REFLECTION.md

## Decisions

One of the most important decisions I made during this project was choosing to create an online store for digital products instead of a more complicated website. I wanted to build something simple but useful for students and people who want to stay organized. I decided to focus on products like planners, journals, templates, and wallpapers because these are common digital products that people can easily download and use.

Another important decision was keeping the design clean and simple. I did not want too many colors or complicated layouts because I wanted users to focus on the products and have an easy experience using the website. I also decided to make the website responsive so it could work on phones, tablets, and computers. Since many people use mobile devices, I thought this was very important.

I also chose to use HTML, CSS, and JavaScript because they are the tools I learned in class and I wanted to improve my skills with basic web development before using advanced frameworks.

---

## What worked

One thing that worked well was the overall design of the website. I am most satisfied with the layout and the organization of the pages. The navigation became much better compared to Project 01, and it became easier for users to move around the website.

I am also happy that I successfully made the website responsive. In the beginning, some sections looked broken on smaller screens, but after testing and fixing the CSS, the website worked much better on mobile devices.

Another thing that worked well was adding product images and creating a more realistic online store experience. Using Canva helped me make the website look more professional. I also liked adding small animations and hover effects because they made the website feel more interactive and modern without making it too complicated.

I am proud that I completed a full website from planning to deployment. Seeing the website live on GitHub Pages made me feel more confident in my web development skills.

---

## What I would do differently

If I started over, I would spend more time planning the structure and design before coding. In the beginning, I changed the layout many times, which caused some confusion and made the code messy in some places.

I would also organize my CSS files better because sometimes it became difficult to find certain styles when fixing problems. Another thing I would change is adding more JavaScript features to make the website more interactive, such as a shopping cart or search feature.

I also think I should have started testing the website on mobile devices earlier instead of waiting until the end. Some responsive design problems took longer to fix because I did not check smaller screens often enough during development.

---

## What I learned

This project taught me a lot about building websites and the full web development process. I learned that creating a website is not only about writing code, but also about planning, design, testing, and improving the user experience.

I improved my skills in HTML and CSS, especially with layouts, spacing, responsive design, and navigation bars. I also became more comfortable using JavaScript for small interactive features.

Another important thing I learned was debugging. Sometimes parts of my website did not work correctly, and I had to spend time finding mistakes in my code. This taught me patience and problem-solving skills.

I also learned more about myself during this project. I realized that I enjoy designing websites and creating ideas for digital products. Even though there were frustrating moments, I felt proud when I solved problems and saw progress in my work.

Overall, this project helped me better understand how websites are created and improved my confidence in web development.
